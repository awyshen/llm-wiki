# 文件处理优化配置说明

## 1. 功能概述

本系统实现了文件导入与处理机制的优化，主要解决以下问题：

1. **相同文件名处理**：自动检测并忽略重复文件名的文件
2. **内容相似度处理**：检测不同文件名但内容相似或相同的文件

## 2. 配置参数

### 2.1 核心配置参数

| 参数名 | 描述 | 默认值 | 说明 |
|--------|------|--------|------|
| file_similarity_threshold | 相似度阈值 | 0.9 | 超过该阈值的文件被视为相似 |
| max_file_size_for_similarity | 最大文件大小 | 10MB | 超过该大小的文件跳过相似度检查 |
| similarity_batch_size | 批量处理大小 | 10 | 数据库查询的批量大小 |

### 2.2 配置文件位置

配置参数可以在系统配置文件中设置，默认位置为：
```
config/config.yaml
```

### 2.3 配置示例

```yaml
# 文件处理优化配置
file_processing:
  # 相似度阈值，范围0-1
  file_similarity_threshold: 0.9
  # 最大文件大小（字节）
  max_file_size_for_similarity: 10485760  # 10MB
  # 批量处理大小
  similarity_batch_size: 10
```

## 3. 使用方法

### 3.1 命令行导入

```bash
# 导入单个文件
python main.py import /path/to/file.txt

# 导入目录
python main.py import /path/to/directory/
```

### 3.2 API导入

使用HTTP POST请求上传文件：

```bash
curl -X POST http://localhost:5050/api/documents/upload \
  -F "file=@/path/to/file.txt"
```

### 3.3 批量导入

```python
from src.collect.file_collector import FileCollector

collector = FileCollector()
results = collector.import_files([
    "/path/to/file1.txt",
    "/path/to/file2.txt",
    "/path/to/file3.txt"
])
print(f"成功导入: {len(results['success'])} 个文件")
print(f"失败导入: {len(results['failed'])} 个文件")
```

## 4. 工作流程

### 4.1 文件导入流程

1. **文件验证**：检查文件是否存在、是否为文件、是否支持的文件类型
2. **文件名检查**：检查是否存在相同文件名的文件
3. **文本提取**：提取文件的文本内容
4. **内容相似度检查**：检查是否存在内容相似的文件
5. **文件复制**：将文件复制到原始文件目录
6. **数据库保存**：将文件信息保存到数据库

### 4.2 重复文件处理流程

1. **相同文件名**：直接跳过导入，返回错误信息
2. **内容相似**：计算与现有文件的相似度，超过阈值则跳过导入

## 5. 日志记录

系统会记录详细的日志信息，包括：

- **文件名重复**：记录重复的文件名和处理结果
- **内容相似**：记录相似文件对、相似度分值及处理结果
- **错误信息**：记录处理过程中的错误信息

### 5.1 日志示例

```
# 文件名重复
2026-04-21 10:09:01,578 - src.collect.file_collector - INFO - 文件 test_file.txt 已存在，跳过导入

# 内容相似
2026-04-21 10:09:31,580 - src.collect.file_collector - INFO - 发现相似文件: test_file.txt (相似度: 1.00)
2026-04-21 10:09:31,603 - src.collect.file_collector - INFO - 文件 similar_file.txt 与现有文件 test_file.txt 内容相似 (相似度: 1.00)，跳过导入
```

## 6. 性能优化

### 6.1 大文件处理

- **跳过大型文件**：超过10MB的文件会跳过相似度检查，提高处理速度
- **分块读取**：计算文件哈希时使用分块读取，避免内存溢出

### 6.2 批量处理

- **批量查询**：从数据库获取文档时使用批量查询，减少数据库交互次数
- **提前返回**：找到相似文件后提前返回，避免不必要的计算

### 6.3 缓存机制

- **文件哈希**：使用MD5哈希值快速比较文件内容，避免重复计算

## 7. 错误处理

系统提供了完善的错误处理机制：

- **文件不存在**：返回FileNotFoundError
- **路径不是文件**：返回ValueError
- **文件已存在**：返回ValueError
- **内容相似**：返回ValueError
- **权限错误**：返回PermissionError

## 8. 最佳实践

### 8.1 配置建议

- **相似度阈值**：根据具体场景调整，一般建议在0.8-0.95之间
- **最大文件大小**：根据服务器内存调整，建议不超过50MB
- **批量处理大小**：根据数据库性能调整，一般建议在5-20之间

### 8.2 使用场景

- **文档管理**：批量导入文档时自动去重
- **内容审核**：检测重复或相似内容
- **数据迁移**：避免重复数据导入

### 8.3 注意事项

- **性能考虑**：处理大量文件时，建议分批导入，避免系统负载过高
- **内存使用**：处理大文件时，注意监控内存使用情况
- **日志管理**：定期清理日志文件，避免磁盘空间不足

## 9. 故障排除

### 9.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 导入失败，提示文件已存在 | 文件名与现有文件相同 | 重命名文件或使用不同的文件名 |
| 导入失败，提示内容相似 | 文件内容与现有文件相似 | 确认是否需要导入相似内容，或修改文件内容 |
| 导入速度慢 | 文件过大或数量过多 | 分批导入，或调整批量处理大小 |
| 内存使用过高 | 处理大文件 | 调整最大文件大小阈值 |

### 9.2 调试建议

- **查看日志**：检查日志文件，了解具体错误信息
- **测试单个文件**：先测试单个文件导入，确认系统正常工作
- **调整配置**：根据实际情况调整配置参数
- **监控资源**：监控系统资源使用情况，避免过载
