# LLM Wiki 系统架构设计文档

## 1. 系统概述

### 1.1 设计目标

LLM Wiki是一个基于Karpathy理念的个人知识管理系统，旨在：

1. **自动知识构建**：从原始文档自动提取和构建结构化知识
2. **知识沉淀**：将知识持久化为可复用的Wiki页面
3. **自我增强**：自动发现知识关联、填补空白、解决矛盾

### 1.2 核心特性

- 多格式文档支持（PDF、Word、Markdown、HTML等）
- AI驱动的知识提取和整理
- 实体识别和知识图谱构建
- 全文检索和智能问答
- 数据备份和恢复

### 2.2 模块划分

#### 2.2.1 核心模块 (core)

- **config**: 配置管理，支持YAML文件和环境变量
- **logger**: 日志管理，支持文件和控制台输出
- **exceptions**: 自定义异常类

#### 2.2.2 采集模块 (collect)

- **file_collector**: 文件采集，支持多种格式
- **web_collector**: 网页采集（预留）
- **watcher**: 文件监控，自动检测新文件

#### 2.2.3 处理模块 (process)

- **llm_client**: LLM客户端，支持OpenAI和Anthropic
- **knowledge_processor**: 知识处理器，协调文档处理流程
- **entity_extractor**: 实体提取器
- **knowledge_graph_builder**: 知识图谱构建器

#### 2.2.4 存储模块 (storage)

- **database**: 数据库管理，SQLAlchemy ORM
- **models**: 数据模型定义
- **wiki_storage**: Wiki文件存储管理
- **backup_manager**: 备份管理

#### 2.2.5 API模块 (api)

- **app**: FastAPI应用
- **schemas**: Pydantic模型

#### 2.2.6 界面模块 (interface)

- **gradio_ui**: Gradio Web界面

## 3. 数据模型

### 3.1 实体关系图

```
┌─────────────┐       ┌─────────────┐       ┌─────────────┐
│  Document   │───────│  WikiPage   │───────│   Entity    │
│  (原始文档)  │       │  (Wiki页面)  │       │   (实体)     │
└─────────────┘       └─────────────┘       └──────┬──────┘
       │                    │                      │
       │                    │                      │
       └────────────────────┴──────────────────────┘
                            │
                     ┌──────┴──────┐
                     │     Tag     │
                     │    (标签)    │
                     └─────────────┘
```

### 3.2 主要数据表

#### Document (原始文档)
- id: 主键
- filename: 文件名
- file_type: 文件类型
- content_hash: 内容哈希
- extracted_text: 提取的文本
- processing_status: 处理状态

#### WikiPage (Wiki页面)
- id: 主键
- title: 标题
- slug: URL标识
- content: 内容
- category: 分类
- file_path: 文件路径

#### Entity (实体)
- id: 主键
- name: 名称
- entity_type: 实体类型
- description: 描述
- aliases: 别名

#### Relation (关系)
- id: 主键
- source_id: 源实体ID
- target_id: 目标实体ID
- relation_type: 关系类型

## 4. 核心流程

### 4.1 文档处理流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  采集   │───>│  解析   │───>│  提取   │───>│  生成   │───>│  存储   │
│Collect  │    │ Parse   │    │Extract  │    │Generate │    │ Store   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
     │              │              │              │              │
     ▼              ▼              ▼              ▼              ▼
 文件采集      文本提取       实体识别      Wiki页面      数据库+文件
 格式检测      元数据解析     摘要生成      关联链接      索引更新
```

### 4.2 知识检索流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  查询   │───>│  解析   │───>│  检索   │───>│  合成   │
│  Query  │    │ Parse   │    │ Search  │    │ Synthesize│
└─────────┘    └─────────┘    └─────────┘    └─────────┘
                    │              │              │
                    ▼              ▼              ▼
                关键词提取      全文搜索       答案生成
                实体识别        实体关联       引用标注
                意图分析        向量检索
```
